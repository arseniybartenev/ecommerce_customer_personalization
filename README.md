## Описание проекта

Интернет-магазин «В один клик» столкнулся со снижением активности постоянных клиентов. Привлечение новых покупателей перестало быть эффективным, поэтому основной упор делается на удержание текущей аудитории. Цель проекта — разработать решение для персонализации предложений, чтобы повысить лояльность и частоту покупок.

**Цель:** Увеличить покупательскую активность путем персонализации предложений для постоянных клиентов. Это включает в себя прогнозирование вероятности снижения активности клиента в ближайшие три месяца и сегментацию пользователей на основе их прибыльности и прогнозируемого поведения.

## Данные

В проекте используются четыре датасета, описывающих поведение клиентов на сайте, их транзакции и прибыльность.

1.  **`market_file.csv`:** Содержит данные о поведении покупателя на сайте, коммуникациях и продуктовых предпочтениях. Ключевые признаки: тип сервиса, согласие на рассылку, маркетинговая активность, длительность регистрации, доля акционных покупок, популярная категория товаров, средний просмотр категорий за визит, количество неоплаченных продуктов, ошибки сервиса, просмотр страниц.
2.  **`market_money.csv`:** Данные о выручке, полученной от покупателя за каждый из последних трех месяцев.
3.  **`market_time.csv`:** Данные о времени (в минутах), проведенном покупателем на сайте за последние два месяца.
4.  **`money.csv`:** Данные о среднемесячной прибыли от продаж каждому покупателю за последние 3 месяца.

## Этапы работы

1.  **Загрузка и первичный анализ данных**
    *   Импорт библиотек и загрузка всех четырех датасетов.
    *   Обзор структуры данных, типов столбцов, поиск явных пропусков и дубликатов.
    *   Выявлены опечатки в категориальных признаках и неверный формат данных в столбце `прибыль`.

2.  **Предобработка данных**
    *   Переименование столбцов для удобства (замена пробелов на нижнее подчеркивание).
    *   Корректировка типов данных (преобразование `прибыль` в числовой формат).
    *   Устранение опечаток в значениях (например, `стандартт` -> `стандарт`, `предыдцщий_месяц` -> `предыдущий_месяц`).

3.  **Исследовательский анализ данных (EDA)**
    *   **Категориальные признаки:** Анализ распределения целевого признака (`покупательская_активность`) выявил дисбаланс классов. Также изучены распределения `тип_сервиса`, `разрешить_сообщать` и `популярная_категория`.
    *   **Количественные признаки:** Построены гистограммы и ящики с усами для таких признаков, как `маркет_актив_6_мес`, `длительность`, `акционные_покупки`, `страниц_за_визит` и других. Анализ проводился в разрезе целевой переменной для выявления различий.
    *   **Выручка и время:** Изучены распределения выручки и времени на сайте по периодам, обнаружены и удалены аномальные выбросы. Также удалены записи клиентов с нулевой выручкой за какой-либо из периодов.

4.  **Объединение таблиц**
    *   Для построения модели все данные были сведены в единый датасет. Таблицы `market_money` и `market_time` были преобразованы с помощью `pivot_table`, чтобы каждый период стал отдельным признаком, и объединены с основной таблицей `market_file` по `id`.

5.  **Корреляционный анализ**
    *   Построена матрица корреляций с использованием библиотеки `phik`, которая позволяет оценивать связи как между числовыми, так и между категориальными признаками. Это помогает понять структуру данных и отобрать наиболее важные признаки для модели.

## Результаты и выводы

В ходе проекта была проведена тщательная предобработка и исследовательский анализ данных интернет-магазина. Были выявлены и устранены аномалии и опечатки, а данные сведены в единый формат, готовый для машинного обучения.

*   **Дисбаланс классов:** Целевая переменная имеет неравномерное распределение, что потребует применения специальных методов (например, взвешивания классов или использования метрик вроде F1) при обучении модели.
*   **Ключевые закономерности:** Анализ распределений количественных признаков в разрезе целевой переменной позволил выявить потенциально важные факторы, влияющие на снижение активности (например, количество ошибок сервиса, доля акционных покупок).
*   **Дальнейшие шаги:** Следующими этапами будут построение пайплайнов для обучения различных моделей, их оценка, анализ важности признаков и, наконец, сегментация покупателей для разработки персонализированных предложений.

Результатом работы станет готовая к использованию модель, способная прогнозировать риск оттока клиентов и помогать в формировании целевых маркетинговых кампаний.

## Запуск проекта

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/arseniybartenev/ecommerce_customer_personalization.git
    ```
2.  Перейдите в папку проекта:
    ```bash
    cd ecommerce_customer_personalization
    ```
3.  Установите зависимости:
    ```bash
    pip install -r requirements.txt
    ```
4.  Запустите Jupyter Notebook:
    ```bash
    jupyter notebook notebooks/personalization_of_offers_in_one_click.ipynb
    ```
